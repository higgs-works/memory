# Human-Like Memory System

> AI를 위한 인간형 기억 시스템
>
> 현재 상세 구현 기준은 [PRD](./PRD.md)를 따른다.

## 문제 정의

### 현재 AI 메모리 서비스의 한계

기존 메모리 서비스(Supermemory, Mem0 등)는 본질적으로 **검색 엔진**이다.

```
사용자: "그거 뭐였지..."
시스템: "검색어를 입력하세요"
```

**근본적 모순**: 기억이 안 나서 찾는 건데, 찾으려면 이미 알아야 함.

### 검색 기반 접근의 구조적 문제

| 문제 | 설명 |
|-----|------|
| **Cold Start** | 뭘 검색할지 모르면 시작 불가 |
| **Context Blindness** | 현재 상황과 무관하게 키워드 매칭 |
| **Flat Memory** | 모든 기억이 동등하게 취급됨 |
| **Explicit Retrieval** | 명시적 요청 없이는 기억이 표면화되지 않음 |
| **Fragmentation** | 연결된 경험이 조각나서 저장됨 |

## 인간 기억의 특성

### 핵심 메커니즘

| 특성 | 설명 | 현재 AI 메모리 |
|-----|------|--------------|
| **연상 활성화 (Spreading Activation)** | 하나가 떠오르면 관련된 것들이 함께 활성화 | ❌ |
| **맥락 기반 회상 (Context-Dependent Recall)** | 비슷한 상황에서 관련 기억이 자동으로 떠오름 | ❌ |
| **망각 곡선 (Forgetting Curve)** | 안 쓰면 희미해지고, 자주 쓰면 강화 | ❌ |
| **감정 가중치 (Emotional Weighting)** | 감정적으로 중요했던 건 더 잘 기억 | ❌ |
| **자동 표면화 (Spontaneous Recall)** | 관련 상황에서 의식적 노력 없이 떠오름 | ❌ |
| **에피소드 기억 (Episodic Memory)** | "그때 그 일" 전체가 하나의 단위로 저장 | ❌ |
| **프라이밍 (Priming)** | 선행 자극이 후속 기억 접근을 촉진 | ❌ |

### 기억의 종류

```
┌─────────────────────────────────────────┐
│              장기 기억                   │
├──────────────────┬──────────────────────┤
│   서술 기억       │     절차 기억         │
│  (Declarative)   │   (Procedural)       │
├────────┬─────────┤                      │
│ 에피소드│ 의미    │   "어떻게" 하는지     │
│ "언제"  │ "무엇"  │   자전거 타기 등      │
│ 경험    │ 사실    │                      │
└────────┴─────────┴──────────────────────┘
```

## 설계 원칙

### 1. 검색 없는 기억 (Search-Free Memory)

사용자가 명시적으로 검색하지 않아도, 현재 컨텍스트에 관련된 기억이 자동으로 표면화된다.

```
AS-IS: user.search("JWT") → results
TO-BE: system.surface(current_context) → relevant_memories
```

### 2. 연상 네트워크 (Associative Network)

기억은 고립되지 않고, 다른 기억들과 연결되어 네트워크를 형성한다.

```
[Auth 구현] ←──→ [JWT 결정] ←──→ [보안 이슈]
      ↑              ↑              ↑
      │              │              │
 [그날 밤샘]     [팀 회의]      [해킹 뉴스]
```

### 3. 동적 강도 (Dynamic Strength)

기억의 접근성은 고정되지 않고, 사용 패턴에 따라 변화한다.

```
strength = base × recency_decay × access_frequency × salience
```

### 4. 에피소드 중심 (Episode-Centric)

의미 없는 청크가 아닌, 맥락이 있는 경험 단위로 저장한다.

## 핵심 컴포넌트

### 1. Context Analyzer
현재 상황(대화, 코드, 환경)을 분석하여 활성화 신호 생성

### 2. Associative Graph
기억 간 연결을 관리하고 spreading activation 수행

### 3. Memory Store
에피소드 단위 기억 저장, 강도/최근성/중요도 관리

### 4. Surfacing Engine
현재 컨텍스트에 가장 관련 있는 기억을 자동으로 선택

### 5. Decay Manager
시간 경과에 따른 기억 강도 감쇠 및 강화 관리
